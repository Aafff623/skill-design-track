# 会议纪要生成

## 工作流程

### Step 1: 读取并分析转写稿

首先读取完整的文字转写稿，理解会议的整体内容和结构。

### Step 2: 清洗内容

去除以下内容：
- 停顿词：嗯、啊、哦、呃、额、那个、这个等
- 冗余表达：重复词语、无意义连接词
- 与会议无关的内容：闲聊、技术故障记录等

**详细参考**: [patterns.md](references/patterns.md) 中的"停顿词过滤"部分

### Step 3: 提取会议基本信息

识别并提取：
| 信息类型 | 识别方式 |
|----------|----------|
| 会议时间 | 日期、时间表达 |
| 会议地点 | 会议室名称、线上会议等 |
| 参会人员 | 自我介绍、被提及的名字 |
| 记录人 | 如果有明确说明 |

**识别模式**: 参考 [patterns.md](references/patterns.md) 中的"信息提取模式"

### Step 4: 识别会议目标

提取以下信息：
- **会议目的**: 使用信号词识别（"今天会议的目的是..."）
- **背景信息**: 相关的前置会议、文档或上下文

### Step 5: 梳理会议内容

按主题组织讨论内容：

1. **识别主题**: 寻找主题转换信号词
2. **提取讨论要点**: 每个主题下的关键内容
3. **记录各方观点**: 按发言人归类观点
4. **总结结论**: 每个主题的讨论结果

**处理原则**:
- 同一观点只记录一次
- 用简洁语言概括长篇发言
- 不确定的内容标注"[需确认]"

### Step 6: 提取行动项

识别并整理：
- **任务描述**: 具体要做什么
- **负责人**: 谁负责执行
- **截止时间**: 何时完成（相对时间转换为具体日期）

### Step 7: 生成输出

使用模板生成结构化会议纪要。

## 模板

会议纪要模板位于: [assets/template.md](assets/template.md)

输出时：
1. 将提取的信息填入对应章节
2. 保持模板的Markdown结构
3. 不确定的内容留空或标注"[需确认]"
