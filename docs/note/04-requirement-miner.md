# 04｜需求清洗（product-requirement-miner）

> 从海量用户评论中挖掘可执行的产品需求，4 步流程 + 人在环路中。

## 核心概念

### 4 步流程

读取 CSV → 清洗分类 → 用户筛选 → 聚类分析 → 生成路线图

### 四类分类标准

| 类别 | 定义 | 信号词 |
|------|------|--------|
| Bug | 功能错误、故障、崩溃 | 闪退、报错、卡死 |
| 建议 | 现有功能的改进意见 | 希望、建议、可以 |
| 需求 | 希望新增的全新功能 | 需要、想要、加 |
| 其他 | 咨询、评价、非功能相关 | 价格、客服 |

### 聚类规则

- 语义相似度 > 85% → 同一簇
- 每簇选信息量最全的作为「基准项」

### 优先级

| 级别 | 定义 |
|------|------|
| P0 | 紧急必做，影响核心功能 |
| P1 | 重要优先，高价值/中成本 |
| P2 | 常规优化 |
| P3 | 远期规划 |

## 关键创新点

「人在环路中」— 分类完让用户选择关注哪类，而非全自动跑完，避免浪费算力处理用户不关心的数据。

## 竞赛价值

- 非技术人员可直接使用
- 展示真实评论数据的处理结果截图，说服力强
