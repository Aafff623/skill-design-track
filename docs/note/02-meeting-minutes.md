# 02｜会议纪要生成（meeting-minutes）

> 7 步工作流，从原始会议内容提取行动项、决策和停顿词过滤。

## 核心概念

### 7 步工作流

读取内容 → 停顿词过滤 → 角色分离 → 关键信息提取 → AskUserQuestion 确认 → 输出结构化纪要 → 行动项追踪

### 输出结构

| 章节 | 内容 |
|------|------|
| 基本信息 | 时间、地点、参会人 |
| 讨论摘要 | 每人观点的简洁复述 |
| 决策记录 | 已明确的决定 |
| 行动项 | 具体任务 + 负责人 + 截止日期 |
| 未决事项 | 待跟进的问题 |

### 关键机制

- **停顿词过滤**：去除「嗯」「啊」「这个」等无意义填充词
- **角色分离**：区分「发言人」和「内容」，保留发言者身份
- **AskUserQuestion**：对模糊决策主动追问，不假设

## 竞赛价值

- 非技术人员直接可用，受众广
- 展示「原始会议内容 → 结构化纪要」前后对比，效果直观
