# 05｜文档工厂（prd）

> Refine-Before-Write 模式，模糊想法 → 完整 PRD 文档（8段式）。

## 核心概念

### 3 阶段工作模式

需求诊断（缺核心信息→追问） → 逻辑构建（流程+状态机） → 细节补全（变态级边界扫描）

### 追问触发条件

缺少以下任一 → 必须先追问，禁止直接生成：
- 核心业务逻辑
- 目标用户
- 成功指标（KPI）

### PRD 8 段式输出

| 章节 | 内容 |
|------|------|
| 1. 业务出发点 | 背景/痛点、核心指标、目标用户 |
| 2. 术语定义 | 统一名词，消除歧义 |
| 3. 用户故事 | 作为...我想要...以便... + 验收标准 |
| 4. 功能清单 | 三级结构表格（模块/子功能/优先级/迭代版本） |
| 5. 逻辑框架 | Mermaid 流程图 + 状态机转换表 |
| 6. 功能详情与边界 | 正常路径 + 5 类边界异常 |
| 7. 技术约束 | 响应时间/QPS/安全性 + 旧数据兼容 |
| 8. 数据采集 | 埋点清单（事件名/触发时机/参数） |

### 变态级边界覆盖

断网 / 高并发 / 权限冲突 / 极端输入 / 版本不兼容

### 写作约束

- 拒绝形容词堆砌（「优秀的」「完善的」一律不用）
- 拒绝废话（每句话必须传递信息）
- 遇到模糊点 → AskUserQuestion，不猜测

## 竞赛价值

- 受众广（每个 PM/创业者都需要）
- 展示「模糊想法 → 完整 PRD」前后对比，效果直观
