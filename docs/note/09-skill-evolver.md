# 09｜进化管家（skill-evolver + weekly-report）

> 4 步流程把对话经验沉淀回 Skill，实现持续迭代优化。

## 核心概念

### skill-evolver：4 步自动进化流程

语料复盘 → 冲突裁决 → 整合优化 → 进化复盘

### Step 1：语料复盘 — 4 维提取

| 维度 | 内容 |
|------|------|
| 隐含约束 | 用户没明说但反复体现的需求 |
| 风格要求 | 输出格式、语气、详略标准 |
| 逻辑澄清 | 步骤优先级、处理顺序、决策逻辑 |
| 负面约束 | 明确禁止或需要避免的事项 |

每条必须引用用户原始表述作为证据。

### Step 2：冲突裁决 — 3 个规则

1. **近时性原则**：对话后期的修正 > 前期指令
2. **具体化原则**：特定场景约束 > 通用约束
3. **场景解耦**：无法合并时用 如果...则...；否则... 保留

### Step 3：整合优化 — 铁律

- 强指令词：必须/始终/确保（做的事）；严禁/禁止/切勿（不做的事）
- 新约束嵌入最相关的原有章节，不追加到末尾
- 包含变更日志：新增/修改/删除

### weekly-report v2.0 进化案例

| 原 v1.0 | v2.0 改进 | 触发原因 |
|---------|----------|---------|
| Markdown 富格式 | 纯文本输出 | 用户反馈偏好纯文本 |
| 使用 emoji | 严禁任何 emoji | 用户明确要求 |
| 允许 XX 占位符 | 强制量化，估算要标注 | 用户要求具体数字 |
