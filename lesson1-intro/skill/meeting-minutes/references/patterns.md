# 会议内容识别模式指南

## 停顿词过滤

### 常见停顿词列表
**语气词/填充词**
- 嗯、啊、哦、呃、额、那个、这个、那个什么、就是说
- 就、然后、那、那么、对、是的、好的、OK、嗯嗯

**冗余表达**
- 重复词语（如"就是就是"、"对对"）
- 无意义的连接词（如"然后呢"、"就是那个"）

**处理原则**
- 完全删除不影响理解的停顿词
- 保留表达逻辑关系的连接词（如"因为"、"所以"、"但是"、"然而"）

## 信息提取模式

### 1. 会议基本信息识别

**时间信息模式**
```
- "今天是XX月XX日" → 提取日期
- "下午两点开始" → 提取具体时间
- "会议时长两小时" → 可用于推算结束时间
```

**地点信息模式**
```
- "在XX会议室" → 提取地点
- "线上会议"、"Zoom会议" → 记录为线上会议
```

**参会人员识别**
```
- 自我介绍："我是XX部门的XX" → 记录姓名和部门
- 被点名："XX说一下" → 记录发言者
- 称呼变化："张总"、"李工" → 尝试关联到具体人员
```

### 2. 会议目标识别

**目的识别信号词**
```
- "今天会议的目的是..."
- "我们主要讨论..."
- "召集大家是为了..."
- "会议目标是..."
```

**背景信息信号**
```
- "上次会议提到..."
- "根据XX文档..."
- "基于..."
- "关于XX项目..."
```

### 3. 主题识别与分类

**主题转换信号**
```
- "接下来我们讨论..."
- "第二个议题是..."
- "关于XX，我来说一下..."
- "我们转到下一个话题"
```

**子主题/要点识别**
```
- 列举："第一、第二、第三..."
- 递进："还有...另外...此外..."
- 转折："但是...不过..."
```

**观点归属识别**
```
- "我认为/我觉得/我建议" → 个人观点
- "我们团队/我们部门" → 集体观点
- "大家都同意" → 共识观点
```

### 4. 行动项提取

**任务分配模式**
```
- "XX负责..." → XX是负责人
- "请XX在XX时间前完成" → 有截止时间
- "需要XX去做..." → 待办事项
```

**时间表达识别**
```
- "今天下午"、"明天" → 相对时间，需转换为具体日期
- "本周内"、"下周一" → 需要结合会议日期推算
- "尽快"、"抓紧时间" → 标注为[待确认具体时间]
```

**任务描述提取**
```
- 动词+对象："完成XX报告"、"跟进XX客户"
- 完整性判断：确保任务描述包含"做什么"和"交付标准"
```

## 不确定内容处理

### 需要确认的情况

1. **信息缺失**
   - 无法确定参会人全名 → 标注"XX（需确认姓名）"
   - 时间模糊 → 标注"[需确认具体时间]"

2. **歧义表达**
   - "稍后"、"过几天" → 标注"[需确认具体日期]"
   - "相关部门" → 标注"[需确认具体部门]"

3. **音频质量问题**
   - 人名发音不清 → 标注"[音频不清晰]"
   - 关键信息缺失 → 标注"[内容缺失]"

### 内容推断规则

**允许推断的情况**
- 明确的称呼（"张总"）可以记录为"张总[需确认全名]"
- 根据上下文可以合理推断的结论

**禁止推断的情况**
- 不要编造不存在的信息
- 不要臆测时间、地点、人员

## 输出格式规范

### 清理原则
- 删除所有停顿词和无关内容
- 保留核心信息和逻辑关系
- 同一观点只记录一次，去重
- 用简洁语言概括长篇发言

### 结构化要求
- 每个主题独立分段
- 观点按发言人归类
- 行动项表格化呈现
- 层次清晰，便于阅读
